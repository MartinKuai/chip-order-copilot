# 业务验收标准

## 概述

本文档定义了电子元器件订单执行风控 Copilot MVP 的验收标准，用于判断项目是否达到交付要求。

## 一、功能验收标准

### 1.1 输入功能

| 编号 | 验收项 | 验收标准 | 优先级 |
|------|--------|----------|--------|
| F-001 | 聊天记录输入 | 支持粘贴多行文本，最大长度 10000 字符 | P0 |
| F-002 | 预设场景切换 | 提供 3-4 个场景按钮，点击后自动填充聊天记录和数据 | P0 |
| F-003 | 场景可修改 | 填充后用户可手动修改聊天记录 | P0 |
| F-004 | CSV 文件上传 | 支持上传 CSV 格式的订单/库存/报价数据 | P0 |
| F-005 | JSON 文件上传 | 支持上传 JSON 格式的订单/库存/报价数据 | P0 |
| F-006 | 文件格式校验 | 上传错误格式文件时给出友好提示 | P0 |
| F-007 | 数据源切换 | 可在内置数据和上传数据之间切换 | P0 |

### 1.2 AI 抽取功能

| 编号 | 验收项 | 验收标准 | 优先级 |
|------|--------|----------|--------|
| A-001 | 型号提取 | 从聊天记录中准确提取物料型号（如 A133） | P0 |
| A-002 | 数量提取 | 准确提取需求数量（如 5000 颗） | P0 |
| A-003 | 特殊要求提取 | 提取标签、包装、混批等特殊要求 | P0 |
| A-004 | 交期提取 | 提取交期紧急程度 | P0 |
| A-005 | 缺失信息识别 | 识别聊天记录中缺失的关键信息 | P0 |
| A-006 | 多型号支持 | 支持一次提取多个型号 | P1 |
| A-007 | 结果展示 | AI 抽取结果以结构化方式展示 | P0 |

### 1.3 风控规则功能

| 编号 | 验收项 | 验收标准 | 优先级 |
|------|--------|----------|--------|
| R-001 | 库存不足检测 | 需求数量 > 可用库存时触发 HIGH 风险 | P0 |
| R-002 | 批次混批检测 | 要求不可混批但库存为散批时触发风险 | P0 |
| R-003 | 报价过期检测 | 报价超过有效期时触发 HIGH 风险 | P0 |
| R-004 | 价格不匹配检测 | 沟通价格与台账价格不一致时触发风险 | P1 |
| R-005 | PO 缺失检测 | 没有正式采购订单时触发 MEDIUM 风险 | P0 |
| R-006 | 标签缺失检测 | 要求贴标但无模板时触发 HIGH 风险 | P0 |
| R-007 | 地址未确认检测 | 收货地址未确认时触发风险 | P1 |
| R-008 | 规则独立运行 | 规则引擎可在 LLM 不可用时独立给出分析 | P0 |

### 1.4 输出功能

| O-003 | 风险来源标注 | 每个风险标注来源（RULE/LLM） | P0 |
| O-004 | Checklist 展示 | 显示发货前待办事项清单 | P0 |
| O-005 | 交接卡片生成 | 生成完整的 Markdown 交接卡片 | P0 |
| O-006 | 一键复制 | 交接卡片支持一键复制到剪贴板 | P0 |
| O-007 | 复制格式兼容 | 复制内容在企微/飞书中排版清晰 | P0 |

## 二、场景验收标准

### 场景 1：常规缺货场景

**输入**:
- 聊天记录："客户：A133要5000颗，最好明天发。销售：库存有，让助理建单。"
- 数据源：内置缺货场景数据

**预期输出**:
- [ ] AI 提取：型号 A133，数量 5000，交期 HIGH
- [ ] 风险触发：🔴 库存不足（需 5000，仅有 3000）
- [ ] 风险触发：🔴 缺少正式 PO
- [ ] 交接卡片：包含库存风险提示和行动建议

### 场景 2：特殊标签场景

**输入**:
- 聊天记录："客户：这批货外箱要贴我们自己的标签，不能混批，上次那个格式。"
- 数据源：内置标签场景数据

**预期输出**:
- [ ] AI 提取：特殊要求 - 贴客户标签、不可混批
- [ ] 风险触发：🔴 标签模板缺失
- [ ] 风险触发：⚠️ 需确认混批策略
- [ ] 交接卡片：包含标签风险和确认事项

### 场景 3：报价过期场景

**输入**:
- 聊天记录："客户：就按上次那个报价 $1.2 来吧，尽快发货。"
- 数据源：内置报价过期场景数据

**预期输出**:
- [ ] AI 提取：价格 $1.2
- [ ] 风险触发：🔴 报价已过期（超过 30 天有效期）
- [ ] 交接卡片：提示需重新核价

### 场景 4：用户自定义场景

**输入**:
- 用户粘贴自己的聊天记录
- 用户上传 CSV/JSON 文件

**预期输出**:
- [ ] AI 正确提取用户聊天记录中的信息
- [ ] 使用用户上传的数据进行风控分析
- [ ] 输出符合用户数据的风险评估结果

## 六、验收流程

1. **自测阶段**: 开发完成所有功能后，按检查清单逐项自测
2. **修复阶段**: 修复自测发现的问题
3. **演示准备**: 准备 3-4 个场景的演示流程
4. **最终验收**: 按验收标准逐项确认
5. **部署上线**: 部署到 Cloudflare Pages

## 七、已知限制

1. **LLM 依赖**: 需要网络连接才能使用 AI 抽取功能（Mock 模式除外）
2. **CF 超时**: Cloudflare Functions 10 秒超时限制
3. **文件大小**: 上传文件最大 1MB
4. **数据持久化**: 刷新页面数据丢失（设计如此）
5. **并发限制**: 免费版每分钟 10 次请求限制
