# MVP 功能范围定义

## 核心原则

本 MVP 的目标是构建一个**可演示、可交互、可解释**的 AI 风控 Copilot，而非一个纯 Mock 展示或普通聊天机器人。

## 功能矩阵

### ✅ 必须实现 (P0 - MVP 核心)

| 功能模块 | 功能点 | 说明 |
|---------|--------|------|
| **输入层** | 沟通记录文本输入 | 支持用户粘贴企业微信/微信聊天记录 |
| **输入层** | 预设场景切换 | 3-4 个典型业务场景一键填充 |
| **数据层** | 内置 Mock 数据 | 订单台账、库存台账、报价记录各 3 套 |
| **数据层** | CSV 文件上传 | 支持用户上传订单/库存/报价 CSV 文件 |
| **数据层** | JSON 文件上传 | 支持用户上传订单/库存/报价 JSON 文件 |
| **数据层** | 数据源切换 | 下拉选择当前激活的数据源 |
| **AI 层** | LLM 字段抽取 | 从聊天记录提取型号、数量、特殊要求等 |
| **AI 层** | 抽取结果展示 | 展示 AI 提取的结构化数据，支持人工修正 |
| **规则层** | 库存风控 | 检查库存是否满足订单需求 |
| **规则层** | 报价风控 | 检查报价是否过期、价格是否匹配 |
| **规则层** | PO 风控 | 检查是否有正式采购订单 |
| **规则层** | 标签风控 | 检查是否有标签模板、是否可混批 |
| **输出层** | 风险可视化 | 红绿灯 + 风险详情列表 |
| **输出层** | 发货 Checklist | 可勾选的待办事项清单 |
| **输出层** | 交接卡片生成 | Markdown 格式，支持一键复制 |
| **兜底层** | Mock 模式开关 | API 失败时可切换到预置结果 |
| **兜底层** | 错误友好提示 | 不崩溃，显示降级提示 |

### ⚠️ 可选实现 (P1 - 如果时间允许)

| 功能模块 | 功能点 | 说明 |
|---------|--------|------|
| **AI 层** | 抽取结果编辑 | 用户可手动修正 AI 提取的字段 |
| **数据层** | 数据预览表格 | 上传后可预览 CSV/JSON 内容 |
| **输出层** | 风险等级徽章 | 高危/警告/安全 的视觉化展示 |

### ❌ 明确不做 (Out of Scope)

| 功能模块 | 功能点 | 原因 |
|---------|--------|------|
| **认证** | 用户登录注册 | MVP 无需用户体系 |
| **认证** | RBAC 权限管理 | MVP 无需权限控制 |
| **持久化** | 数据库存储 | 纯展示 Demo，刷新可丢失 |
| **ERP** | 真实 ERP 对接 | 避免陷入庞大系统 |
| **ERP** | 自动下单 | 仅做风控提示，不做执行 |
| **ERP** | 自动分货 | 仅做风险提示，不做决策 |
| **报价** | 自动报价 | 仅检查报价有效性，不做定价 |
| **审批** | 真实审批流 | 仅生成建议，不做流转 |

## Demo 演示流程

### 场景 1：常规缺货场景
1. 用户点击"缺货场景"按钮
2. 沟通记录自动填充：客户要 5000 颗 A133，明天发货
3. 系统自动挂载库存数据：A133 仅有 3000 颗
4. 点击"执行风控分析"
5. AI 提取：型号 A133，数量 5000，交期紧急
6. 规则引擎触发：🔴 库存不足（需 5000，仅有 3000）
7. 生成交接卡片：包含风险提示和行动建议

### 场景 2：特殊标签场景
1. 用户点击"标签场景"按钮
2. 沟通记录自动填充：客户要求贴专属标签，不可混批
3. 系统自动挂载订单数据：标签模板不可用
4. 点击"执行风控分析"
5. AI 提取：特殊要求 - 贴客户标签、不可混批
6. 规则引擎触发：🔴 标签模板缺失、⚠️ 需确认混批策略
7. 生成交接卡片：包含标签风险和确认事项

### 场景 3：报价过期场景
1. 用户点击"报价过期"按钮
2. 沟通记录自动填充：按之前的报价 $1.2 执行
3. 系统自动挂载报价数据：上次报价已过期 30 天
4. 点击"执行风控分析"
5. AI 提取：价格 $1.2
6. 规则引擎触发：🔴 报价已过期，需重新核价
7. 生成交接卡片：包含价格风险和核价建议

### 场景 4：用户自定义场景
1. 用户手动粘贴自己的沟通记录
2. 用户上传 CSV/JSON 文件或选择内置数据
3. 点击"执行风控分析"
4. 系统展示完整分析结果

## 验收标准

### 功能验收
- [ ] 输入不同沟通记录，AI 能准确提取不同型号和数量
- [ ] 缺少标签模板时，风险面板亮红灯
- [ ] 报价过期时，系统提示需重新核价
- [ ] 缺少 PO 时，交接状态为"等待确认"
- [ ] 库存不足时，触发缺货警报
- [ ] 要求不可混批但库存为散批时，系统拦截
- [ ] API 失败时，可无缝切换到 Mock 模式
- [ ] 交接卡片可完美复制到企微/飞书

### 体验验收
- [ ] 页面加载时间 < 3 秒
- [ ] 分析结果返回时间 < 10 秒（Mock 模式 < 1 秒）
- [ ] UI 视觉专业，非简陋毕设风格
- [ ] 操作流程清晰，无需额外说明
