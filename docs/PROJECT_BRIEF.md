# 电子元器件订单执行风控 Copilot - 项目简介

## 项目定位

**产品名称**: 电子元器件订单执行风控 Copilot

**核心价值定位**: 面向芯片代理商 B2B 订单执行场景的 AI 协同助手。通过 LLM 将销售、客户、助理之间非结构化的碎片沟通记录，转化为结构化的订单交接卡片，并结合结构化台账（订单、库存、报价）与预设业务规则进行风控校验。

**目标达成**: 帮助销售助理在执行发货前，自动识别信息缺口（如：缺失标签、无正式PO）和业务风险（如：报价过期、库存被占），从而减少错发漏发、避免合规争议和内部扯皮。

## 用户画像

### 核心用户：销售助理 (Sales Assistant)
- **痛点**: 经常在信息黑盒中接手任务，对历史沟通不知情；容易遗漏后置的特殊要求；无权限确认价格/库存状态；出了问题容易成为"背锅侠"
- **诉求**: 明确当前订单能不能发、缺什么条件、风险在哪里

### 上游用户：销售 (Sales)
- **痛点**: 在外应酬或开会，没时间填复杂的审批单或交接表
- **诉求**: 复制粘贴几句聊天记录就能把活交接出去

### 管理用户：销售经理 / 商务负责人
- **诉求**: 订单流转透明化，系统级规避价格倒挂和违规发货风险

### 下游用户：库房 / 物流
- **诉求**: 只看明确的指令（贴什么签、怎么包装、发哪里）

## 核心业务流程

1. **信息输入**: 销售或助理将企业微信/微信中的多方群聊记录复制粘贴至 Copilot
2. **上下文挂载**: 助理选择或上传对应的结构化业务数据（订单台账、库存台账、报价记录）
3. **AI 抽取**: LLM 从杂乱的对话中提取关键业务实体（型号、数量、特殊包装/标签要求、交期期望等）
4. **规则融合校验**: 系统将 AI 提取的内容与结构化台账进行 Join，通过本地业务规则引擎进行逻辑判断（如：提取到价格为 $1.2，校验台账发现报价已过期）
5. **卡片生成与协同**: Copilot 输出带红绿灯状态的风险报告、发货 Checklist，并生成一份规范的 Markdown/文本格式交接卡片
6. **分发**: 助理一键复制卡片，发送至内部执行群，各环节按卡片指令行事

## 技术架构概览

- **前端**: React + Vite + 原生 CSS/CSS Modules
- **后端**: Cloudflare Pages Functions (Serverless)
- **AI**: LLM API 调用（小米 MiMo API 或其他模型）
- **部署**: Cloudflare Pages（零维护成本）

## 项目边界

### In Scope (第一版必须包含)
- 纯文本聊天记录输入框
- 内置多套高仿真的 JSON Mock 数据源，支持下拉切换
- **CSV/JSON 文件上传功能**（支持用户上传自有数据）
- 基于 LLM 的非结构化字段抽取能力
- 纯前端/Serverless 层面的规则引擎判断逻辑
- 风险可视化面板（红绿灯、Checklist）
- 结构化交接卡片生成及剪贴板复制功能
- 完善的错误兜底逻辑（Mock 模式切换）

### Out of Scope (绝对不碰)
- 任何真实的 ERP/CRM 系统 API 对接
- 用户登录注册、RBAC 权限管理
- 数据库持久化存储（刷新页面可丢失数据，纯展示用）
- 真实的审批流流转动作（仅生成建议）
- 复杂的自动分货、自动锁库算法（仅基于当前状态做风险提示）
- 自动报价功能
- 自动下单功能

## 项目目标

1. **可演示**: 面试时能流畅展示完整的风控分析流程
2. **可交互**: 用户可以粘贴自己的沟通记录、上传 CSV/JSON 数据
3. **可解释**: 每个风险判断都有明确的依据和来源
4. **可部署**: 一键部署到 Cloudflare Pages，无需后端运维
