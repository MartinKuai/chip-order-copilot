# 开发前风险清单

## 概述

本文档梳理了电子元器件订单执行风控 Copilot 项目在开发过程中可能遇到的风险，以及对应的缓解措施。

## 一、纯 Mock 展示风险

### 风险描述
项目可能退化为一个纯 Mock 展示，用户无法输入自己的数据，所有结果都是预置的，失去"可交互"的核心价值。

### 风险表现
- 所有分析结果都是预置的，与用户输入无关
- 用户只能看不能玩，无法验证不同场景
- 演示时被质疑"这不是 AI，是写死的"

### 缓解措施
1. **必须实现 CSV/JSON 上传功能**: 用户可上传自有数据
2. **AI 抽取必须基于真实 LLM**: 不是预置结果
3. **规则引擎必须动态计算**: 基于用户数据实时计算
4. **验收标准明确**: 输入不同数据必须产生不同结果

### 检查点
- [ ] 用户可粘贴自己的聊天记录
- [ ] 用户可上传 CSV/JSON 文件
- [ ] AI 抽取结果与输入内容相关
- [ ] 风险评估结果随数据变化

---

## 二、普通聊天机器人风险

### 风险描述
项目可能做成一个普通聊天机器人，只是简单的问答交互，失去"风控工作台"的专业定位。

### 风险表现
- UI 设计成对话式界面
- 缺乏结构化的风控面板
- 没有红绿灯、Checklist 等专业元素
- 与 ChatGPT 等通用工具无差异

### 缓解措施
1. **坚持工作台布局**: 左右分栏，不是对话式
2. **强化风控视觉**: 红绿灯、风险列表、Checklist
3. **突出专业价值**: 展示规则引擎、结构化输出
4. **交接卡片是核心交付物**: 不是聊天回复

### 检查点
- [ ] UI 是工作台布局，不是聊天界面
- [ ] 有明显的风控面板（红绿灯）
- [ ] 有 Checklist 待办事项
- [ ] 输出是结构化交接卡片

---

## 三、CSV/JSON 上传踩坑点

### 风险描述
文件上传功能可能遇到各种技术问题，影响用户体验。

### 可能问题

#### 3.1 格式解析失败
- **问题**: CSV 编码问题（UTF-8/GBK）、JSON 格式错误
- **缓解**:
  - 前端预校验文件格式
  - 提供清晰的格式要求说明
  - 支持 UTF-8 BOM 自动检测
  - 错误提示具体到行号

#### 3.2 字段映射问题
- **问题**: 用户上传的字段名与预期不一致
- **缓解**:
  - 提供 CSV 模板下载
  - 支持字段名模糊匹配
  - 提供字段映射配置界面

#### 3.3 文件大小限制
- **问题**: CF Functions 请求体限制 10MB
- **缓解**:
  - 前端限制文件大小（最大 1MB）
  - 大文件分片上传（MVP 不实现）
  - 提示文件过大时的处理方案

#### 3.4 数据验证
- **问题**: 上传的数据不符合业务逻辑
- **缓解**:
  - 前端基础校验（必填字段、数据类型）
  - 后端业务校验（库存数量 > 0、价格 > 0）
  - 校验失败时给出明确提示

### 检查点
- [ ] CSV 文件可正常解析（UTF-8/GBK）
- [ ] JSON 文件格式错误时有友好提示
- [ ] 文件大小超限时有明确提示
- [ ] 字段缺失时有具体错误信息
- [ ] 提供了 CSV 模板下载

## 四、LLM API 失败兜底策略

### 风险描述
LLM API 可能因网络、超时、额度等原因不可用，影响核心功能。

### 兜底策略

#### 4.1 自动降级
```
LLM 调用
    │
    ├── 成功 → 使用真实结果
    │
    └── 失败
            │
            ▼
        记录错误日志
            │
            ▼
        自动切换到 Mock 模式
            │
            ▼
        返回预置结果（标注 mock_mode: true）
            │
            ▼
        前端显示"已切换到预置结果"
```

#### 4.2 手动切换
- 提供 Mock 模式开关
- 用户可主动切换到 Mock 模式
- 适合网络不稳定时的演示场景

#### 4.3 预置场景匹配
- 根据用户输入的关键词匹配预置场景
- 关键词：缺货、标签、报价、PO
- 匹配失败时使用通用场景

#### 4.4 错误分类处理

| 错误类型 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 网络超时 | 自动降级 | "AI 分析超时，已使用预置结果" |
| API 错误 | 自动降级 | "AI 服务暂时不可用，已使用预置结果" |
| 解析失败 | 自动降级 | "AI 返回结果异常，已使用预置结果" |
| 额度用尽 | 自动降级 | "AI 额度已用完，已使用预置结果" |

### 检查点
- [ ] LLM 超时后自动降级
- [ ] LLM 错误后自动降级
- [ ] 降级时有友好提示
- [ ] Mock 模式开关可用
- [ ] 预置场景匹配逻辑正常

## 五、Cloudflare Pages Functions 部署问题

### 风险描述
CF Pages Functions 有诸多限制，可能导致部署或运行问题。

### 可能问题

#### 5.1 超时限制
- **问题**: 免费版 10 秒运行时间限制
- **影响**: LLM API 调用可能超时
- **缓解**:
  - LLM 调用设置 8 秒超时
  - 前端设置 10 秒总超时
  - 超时后自动降级到 Mock 模式

#### 5.2 请求体限制
- **问题**: 请求体最大 10MB
- **影响**: 大文件上传可能失败
- **缓解**:
  - 前端限制文件大小（最大 1MB）
  - 压缩请求数据
  - 分片上传（MVP 不实现）

#### 5.3 冷启动延迟
- **问题**: Functions 首次调用可能有延迟
- **影响**: 用户体验不佳
- **缓解**:
  - 显示加载动画
  - 预热 Functions（定期 ping）
  - 使用 Websocket（MVP 不实现）

#### 5.4 环境变量配置
- **问题**: LLM_API_KEY 等配置错误
- **影响**: LLM 调用失败
- **缓解**:
  - 提供配置检查接口
  - 启动时验证环境变量
  - 提供 Mock 模式作为备选

#### 5.5 路由配置
- **问题**: 前端路由和 API 路由冲突
- **影响**: 页面 404 或 API 404
- **缓解**:
  - 正确配置 _routes.json
  - API 路由以 /api/ 开头
  - 测试所有路由

#### 5.6 日志和调试
- **问题**: Functions 日志查看不便
- **影响**: 问题排查困难
- **缓解**:
  - 使用 console.log 记录关键信息
  - 通过 CF Dashboard 查看日志
  - 提供本地开发环境

### 检查点
- [ ] LLM 调用超时设置正确（8秒）
- [ ] 请求体大小限制已配置
- [ ] 环境变量配置正确
- [ ] 路由配置正确
- [ ] 可通过 CF Dashboard 查看日志

## 六、范围膨胀风险

### 风险描述
第一版开发过程中可能出现功能范围膨胀，导致项目无法按时交付。

### 必须避免的范围膨胀

#### 6.1 认证系统
- ❌ 不做用户登录注册
- ❌ 不做 RBAC 权限管理
- ❌ 不做多租户支持

#### 6.2 持久化存储
- ❌ 不接数据库
- ❌ 不做数据持久化
- ❌ 不做历史记录

#### 6.3 ERP 集成
- ❌ 不接真实 ERP
- ❌ 不做自动下单
- ❌ 不做自动分货
- ❌ 不做库存锁定

#### 6.4 报价系统
- ❌ 不做自动报价
- ❌ 不做价格计算
- ❌ 不做报价审批

#### 6.5 审批流
- ❌ 不做真实审批流
- ❌ 不做消息通知
- ❌ 不做状态流转

#### 6.6 过度优化
- ❌ 不做过早性能优化
- ❌ 不做复杂缓存策略
- ❌ 不做微服务拆分

### 应该做的
- ✅ 聚焦风控分析核心功能
- ✅ 保证 Mock 模式完整可用
- ✅ 保证规则引擎可独立运行
- ✅ 保证交接卡片可正常复制
- ✅ 保证错误兜底机制正常

### 检查点
- [ ] 未实现用户认证系统
- [ ] 未接入数据库
- [ ] 未接真实 ERP
- [ ] 未实现自动报价
- [ ] 未实现审批流
- [ ] 功能范围与 MVP_SCOPE.md 一致

---

## 七、其他潜在风险

### 7.1 演示环境风险
- **问题**: 面试现场网络不稳定
- **缓解**: 提前准备 Mock 模式演示

### 7.2 LLM 输出不稳定
- **问题**: LLM 每次输出可能不同
- **缓解**:
  - 规则引擎兜底
  - 允许用户修正抽取结果

### 7.3 视觉效果不达标
- **问题**: UI 看起来像简陋毕设
- **缓解**:
  - 参考优秀 B 端产品设计
  - 使用现成的 CSS 组件库
  - 重视细节打磨
